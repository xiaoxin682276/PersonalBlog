git国内使用魔法无法推送

发布时间：2025-07-12 | 分类：技术分享 | 作者：昕染

问题描述

当我们使用魔法软件在Git进行代码推送时，经常会遇到以下类似错误：

fatal: unable to access ' 网页链接;: failed to connect to 127.0.0.1 
port 7890 after 2092 ms: couldn't connect to server
⚠️ 注意：这个问题通常是由于代理端口配置不正确导致的。

解决方案

在开始之前，请确保您的代理服务正常运行，且网络流畅

步骤一：查看代理服务器端口

在您的电脑中依次打开设置-->网络和Internet-->代理-->手动设置代理-->编辑

查看代理服务器地址右侧的端口号，先记下来

步骤二：打开cmd终端

依次输入以下命令：

git config --global http.proxy http://127.0.0.1:(端口号)
git config --global https.proxy http://127.0.0.1:(端口号)

步骤三：检验配置是否完成

继续在终端输入以下代码：

git config --global -l
✅ 成功提示：如果看到输出了账户信息，说明配置完成。
